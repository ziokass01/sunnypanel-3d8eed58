Server runtime fix summary

Files changed:
1) supabase/functions/server-app-runtime/index.ts
2) customer-worker/index.js

What was fixed

A. server-app-runtime heartbeat no longer turns stale sessions into noisy 404/409
- Before:
  - action=heartbeat with an old session_token called touchRuntimeSession()
  - if session was missing/inactive/expired, function returned 404 or 409
- After:
  - for SESSION_NOT_FOUND / SESSION_INACTIVE / ENTITLEMENT_INACTIVE / ENTITLEMENT_EXPIRED / ENTITLEMENT_REVOKED
  - function tries to bootstrap a fresh session again when account_ref + device_id are available
  - if no session can be rebuilt, function returns 200 with active=false instead of throwing 404/409
- Result:
  - repeated stale heartbeat calls stop flooding logs with harmless 404/409 noise

B. logout is now idempotent
- Before:
  - logout on an already-missing session returned 404
- After:
  - missing/inactive session returns 200 with already_logged_out=true
- Result:
  - repeated logout / stale token cleanup no longer pollutes logs with 404s

C. customer-worker now forwards x-fp and x-admin-key
- Before:
  - worker CORS allowed headers did not include x-fp / x-admin-key
  - worker also did not forward x-fp to upstream
- After:
  - x-fp and x-admin-key are allowed and forwarded
- Result:
  - runtime bootstrap based on device fingerprint works correctly when requests pass through worker
  - fewer SESSION_BOOTSTRAP_REQUIRED / session mismatch cases

D. customer-worker now normalizes ugly upstream HTML 502 into JSON
- Before:
  - upstream 502 text/html was passed through raw, causing BAD_RUNTIME_JSON style client errors
- After:
  - worker converts upstream HTML 5xx into JSON { code: "UPSTREAM_BAD_GATEWAY" }
- Result:
  - client receives structured JSON instead of Cloudflare / HTML garbage when using the worker

Important note about the 502 in Supabase Invocations
- The specific 502 entries where deployment_id=null and function_id=null are upstream gateway failures before your function code fully runs.
- Repo code cannot completely eliminate those platform-side failures.
- What this patch does is reduce the extra 404/409 churn and make worker-mediated failures return clean JSON.

Deploy order
1) Deploy edge function server-app-runtime
2) Deploy / update customer-worker if you use the worker path
3) Test heartbeat, logout, consume, unlock_feature again

Recommended quick tests
- heartbeat with an old session_token should now return 200 and recover or downgrade cleanly
- logout with an old session_token should now return 200
- requests through worker should preserve x-fp and no longer dump raw HTML 502 into the client
